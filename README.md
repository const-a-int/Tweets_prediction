# Tweets_prediction

# Анализ тональности твитов (Sentiment Analysis)

## Описание проекта

Задача — определить эмоциональную окраску твита: негативная, нейтральная или позитивная.

Проект включает полный цикл работы с текстовыми данными: предобработка, разведочный анализ, векторизация и обучение моделей классификации.

---

## Данные

Три файла с твитами разной тональности:
- `processedNegative.csv` — негативные (метка 1)
- `processedNeutral.csv` — нейтральные (метка 2)
- `processedPositive.csv` — позитивные (метка 3)

Каждый файл изначально имел структуру, где твиты были в индексах. В ходе подготовки данные были трансформированы в удобный формат (столбец `tweet` + столбец `target`).

---

## Этапы работы

### 1. Предобработка текста
- Удаление ссылок 
- Удаление всех символов, кроме букв и пробелов
- Приведение к нижнему регистру
- Токенизация (разбиение на слова)

### 2. Разведочный анализ (EDA)
- Визуализация распределения классов
- Подсчёт общего количества слов и размера словаря
- Анализ длины твитов

**Результаты:**
- Всего слов: 33 208
- Уникальных слов: 6 378
- Максимальная длина твита: 30 слов

### 3. Векторизация
Преобразование текста в числовые признаки с помощью:
- `CountVectorizer` (частотная матрица)
- `TfidfVectorizer` (TF-IDF)
- **Бинаризация** — преобразование признаков в бинарный формат (1, если слово встречается, иначе 0), что полезно для моделей, чувствительных к масштабу частот

### 4. Классификация
Обучались модели:
- Logistic Regression
- Random Forest
- Linear SVC

## Используемые библиотеки

- `pandas`, `numpy` — обработка данных
- `nltk`, `regex` — токенизация и очистка текста
- `sklearn` — векторизация, модели, метрики
- `matplotlib`, `seaborn` — визуализация
- `gensim` — Word2Vec (дополнительно)
